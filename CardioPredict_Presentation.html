The Challenge

Problem Statement

Cardiovascular diseases kill 17.9 million people annually — the #1 cause of death globally.

💰

Expensive Diagnostics

ECG, angiography, and cardiac biomarker tests require specialised equipment and trained specialists.

⏳

Time-Consuming

Traditional diagnostic pathways involve multiple visits, long waiting periods, and delayed results.

🏥

Limited Access

Rural and semi-urban areas lack the specialised infrastructure needed for cardiac diagnostics.

🎯

Our Solution

An ML-powered screening tool using 13 routine clinical parameters for instant risk assessment.

Goals

Project Objectives

  1. Analyse the Cleveland Heart Disease dataset (303 records, 13 features)
  2. Train & compare 7 ML classification algorithms
  3. Build a responsive Flask web application for clinical users
  4. Implement dual-layer input validation (client + server)
  5. Store prediction history using SQLite database
  6. Containerise with Docker for portable deployment
  7. Test with 28 automated pytest test cases

Preprocessing

  • Missing values: Median imputation for ca, thal
  • Scaling: StandardScaler (z-score normalisation)
  • Split: 80/20 stratified train-test split
  • Seed: random_state=42 for reproducibility

Evaluation Strategy

  • Cross-Validation: 10-fold stratified CV
  • Metrics: Accuracy, Precision, Recall, F1
  • Visualisation: Confusion matrix, ROC
  • Comparison: 7 algorithms benchmarked

Key Insight

The model achieves 96.97% Recall — correctly identifying nearly all disease-positive patients. Only 1 false negative in 61 test cases.

In clinical screening, high recall is critical — missing a disease case (false negative) has far more severe consequences than a false alarm (false positive).
